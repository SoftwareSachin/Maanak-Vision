## PART 2 — THE EXACT THINGS THAT MAKE AI UI LOOK LIKE AI UI

These are the tells. Every time an AI generates UI, it does some combination of these. You are banned from doing any of them. Not "try to avoid." Banned.

**BANNED — visual:**
- Any gradient whatsoever. Not subtle. Not "just on the button." Zero gradients.
- Glassmorphism. No frosted glass. No background-blur. No translucent cards.
- Neumorphism. No soft inner shadows. No embossed surfaces.
- Illustration on any screen. No SVG drawings of factories, workers, robots, gears, shields, checkmarks with sparkles.
- Lottie animation or any animated illustration.
- Abstract icons. Every icon must represent a real physical object. A caliper. A ruler. A stamp. A camera. Not: a hexagon with lines, a brain, a chip, a network node, a floating circle with a checkmark.
- Gradient text. Including yellow-to-orange on headings.
- Decorative shapes in the background. No blobs, no circles, no diagonal stripes behind content.
- Rounded cards floating on a lighter background with drop shadows.
- Any hero section with large illustration + headline + subheadline + two buttons.
- Onboarding screens with a full-page illustration taking 60% of the screen.
- Empty state screens with a large centered illustration and a friendly caption.
- A mascot of any kind.
- Confetti, particle effects, fireworks on success.
- Shimmer loading skeletons styled like content placeholders with gradient animation.

**BANNED — copy and language:**
- "Empower your quality journey"
- "Smart insights at your fingertips"
- "AI-powered excellence"
- Any tagline that sounds like a startup pitch.
- Any screen heading that ends in an exclamation mark.
- "Welcome back, Sunil!" as a greeting header.

**BANNED — layout patterns:**
- A centered logo + tagline as the main screen.
- Metric cards with a large number, a small label, and a subtle upward arrow — arranged in a 2x2 grid with rounded corners and drop shadows.
- A sidebar navigation on mobile.
- Tabs styled as pill-shaped toggle buttons with a sliding background indicator.
- A bottom sheet that has a drag handle + illustration + headline + two buttons.
- Step indicators styled as numbered circles with thick colored borders and animated transitions.

## PART 3 — WHAT IT MUST LOOK LIKE INSTEAD

Describe every screen using only these building blocks. Nothing else exists.

**The only allowed background colors:**
- #0f0f0f (near black — primary background)
- #1a1a1a (dark surface — cards, panels)
- #252525 (elevated surface — modals, bottom sheets)
- #F5C518 (Safety Yellow — primary action only)
- #22C55E (Pass green — status only)
- #EF4444 (Fail red — status only)
- #F59E0B (Warning amber — status only)

**The only allowed text colors:**
- #FFFFFF (primary text — headings, values)
- #A1A1A0 (secondary text — labels, metadata)
- #6B6B6B (muted text — timestamps, helper text)
- #F5C518 (yellow — active states, CTAs only)
- #22C55E / #EF4444 / #F59E0B (status text only)

**The only allowed UI components:**

*Rows (use for everything list-related):*
Height: 56dp fixed. Left: 40x40dp icon or thumbnail, no border-radius OR 4dp max radius. Center: primary label 15sp medium + secondary label 12sp gray. Right: value or status badge. Separated by 0.5px border #2a2a2a. No shadow. No card wrapper.

*Status badges (the only allowed decorative element):*
Pill shape. Filled background (not outlined). Green/Red/Amber. White text 11sp uppercase. Padding 4dp vertical 10dp horizontal. No border. No shadow. Used only for Pass/Fail/Warning.

*Buttons (primary):*
Full width. 60dp height. Background #F5C518. Text #000000 Rajdhani Bold 18sp uppercase. Border-radius 6dp only. No shadow. No gradient. No icon unless absolutely necessary.

*Buttons (secondary):*
Full width. 60dp height. Background transparent. Border 1px #F5C518. Text #F5C518 Rajdhani Medium 16sp. No shadow.

*Bottom navigation:*
Background #1a1a1a. 4 tabs maximum. Icon 24dp + label 10sp below. Active tab: icon and label in #F5C518. Inactive: #6B6B6B. No pill background behind active tab. No animation. Top border 0.5px #2a2a2a.

*Top bar:*
Background same as screen (#0f0f0f). Left: back arrow or hamburger 24dp. Center: screen title in Rajdhani 600 18sp white. Right: icon action if needed. No drop shadow. No border. Height 56dp.

*Dividers:*
0.5px solid #2a2a2a. Full width. Used between list rows and between sections. Nothing else.

*Section headers (within a screen):*
12sp uppercase, letter-spacing 1.2px, color #6B6B6B. Left-aligned. Padding 16dp left, 8dp top, 4dp bottom. No background. No border.

## PART 5 — FINAL SANITY CHECK BEFORE OUTPUT

Before you output anything, answer these questions internally:

1. Is there a gradient anywhere? → Remove it.
2. Is there an illustration, SVG drawing, or Lottie animation anywhere? → Remove it.
3. Do the cards have drop shadows and rounded corners floating on a lighter background? → Remove the shadow, flatten the card.
4. Does any screen have a hero section with a tagline? → Remove it.
5. Does any icon represent an abstract concept (AI, intelligence, network, data)? → Replace with a physical real-world object icon.
6. Would a Swiggy engineer look at this and say "yaar, ye toh AI ne banaya hai"? → Start over.
7. Is the information density comparable to PhonePe's transaction history? → If it feels sparse, add more data rows.
8. Are all primary actions reachable by the right thumb without moving the hand? → If not, restructure.

Only output after passing all 8 checks.
